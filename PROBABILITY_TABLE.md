## 確率計算式

### 基本計算式
```
補正後確率 = floor(基本確率 + adjustment)
```

### 調整値(adjustment)の計算

#### 正の幸運時（totalLuck ≥ 0）
**高品質アイテム（Quality > 0）の場合**:
```
diminishingFactor = 1.0 / (1.0 + Quality × factor)
adjustment = Quality × 幸運値 × diminishingFactor
```
**その他のアイテム（Quality ≤ 0）の場合**:
```
adjustment = Quality × 幸運値
```

#### 負の幸運時（totalLuck < 0）
- **Quality < 0** (低品質アイテム): `adjustment = |Quality × 幸運値|` (負の効果を正に反転)
- **Quality > 0** (高品質アイテム): `adjustment = Quality × 幸運値 × 負の幸運倍率`
- **Quality = 0** (中性アイテム): `adjustment = 0`

### 品質値による影響

#### 正の幸運時
- Quality > 0: 幸運値が高いほど有利（**ただし減衰効果あり**）
- Quality = 0: 幸運値の影響なし  
- Quality < 0: 幸運値が高いほど不利

#### 負の幸運時（改良版）
- Quality > 0: 幸運の負の効果が1.5倍に増幅（より不利）
- Quality = 0: 幸運値の影響なし
- Quality < 0: **幸運の負の効果が正に反転**（より有利）

### 特別処理
- **高品質減衰効果**: 正の幸運時、Quality値が高いほど幸運ボーナスが減衰
  - factor = 0.3（デフォルト）
  - Quality 1: 減衰係数 = 1/(1+1×0.3) = 0.77 (23%減衰)
  - Quality 2: 減衰係数 = 1/(1+2×0.3) = 0.63 (37%減衰)
  - Quality 3: 減衰係数 = 1/(1+3×0.3) = 0.53 (47%減衰)
- **負の幸運倍率**: 正のQualityに対して 1.5倍 の負の効果
- **低品質アイテムブースト**: 負のQualityのアイテムは負の幸運で確率上昇
- **幸運値制限**: 最小値 -10.0%、最大値 +10.0%

## 計算式

### 属性タイプ

Minecraftの属性修飾子には3つの操作タイプが存在し、それぞれ異なる計算方法を使用します：

| 属性 | 説明 | 計算式 |
|-----------|------|--------|
| **add_value** | 修飾子の値を基本値に加算 | `合計 = 基本値 + 値1 + 値2 + ... + 値n` |
| **add_multiplied_base** | 基本値に(1 + 修飾子の合計)を乗算 | `合計 = 基本値 × (1 + 値1 + 値2 + ... + 値n)` |
| **add_multiplied_total** | 前段階の合計に(1 + 値)を乗算 | `合計 = 前段階合計 × (1 + 値1) × (1 + 値2) × ... × (1 + 値n)` |

### 計算例

#### add_value の例
基本値3に、値2と値4の2つのadd_value修飾子を適用：
```
合計 = 3 + 2 + 4 = 9
```

#### add_multiplied_base の例
基本値3に、値2と値4の2つのadd_multiplied_base修飾子を適用：
```
合計 = 3 × (1 + 2 + 4) = 3 × 7 = 21
```

#### add_multiplied_total の例
基本値3に、値2と値4の2つのadd_multiplied_total修飾子を適用：
```
合計 = 3 × (1 + 2) × (1 + 4) = 3 × 3 × 5 = 45
```

### 複合計算の順序

Minecraftでは属性修飾子は以下の順序で適用されます：

1. すべての **add_value** 修飾子を基本値に加算
2. すべての **add_multiplied_base** 修飾子を適用（基本値に対して）
3. すべての **add_multiplied_total** 修飾子を順次適用（累積的に）

例：基本値10、add_value(+5)、add_multiplied_base(0.5)、add_multiplied_total(0.2)の場合：
```
段階1: 10 + 5 = 15 (add_value)
段階2: 15 + (10 × 0.5) = 15 + 5 = 20 (add_multiplied_base)
段階3: 20 × (1 + 0.2) = 20 × 1.2 = 24 (add_multiplied_total)
```

#### 釣りの幸運計算での適用（通常時）
